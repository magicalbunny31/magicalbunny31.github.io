definitions

  • service
    • it's what the apps above provide you with: their "service"
  • app/application/bot
    • an application/bot that you are able to authorise and grant access to (for example, access to a guild)
  • application command
    • a command that can be invoked on discord, which is used to provide you with the service
    • this includes: slash commands and context menu commands
  • interaction
    • the response the applications receive when you interact with it through discord
    • this is represented by: invoking an application command, completing a command's autocomplete choices, using buttons/select menus and submitting a modal
  • id
    • discord's way of identifying anything on discord, which is guaranteed to be unique
    • for more information, read discord's developer documentation
  • guild
    • an isolated collection of users and channels, often referred to as "servers" in the ui
    • for more information, read discord's developer documentation
  • user data
    • data about someone which can be used to identify a user on discord
  • usage data
    • data that is collected automatically generated by using the service
  • developers
    • this refers to the developers of the service that is provided
    • these are people who are related to the development process for any of these apps and/or services

processing of data

  • the applications require the following information to be processed through their servers in order to provide you with the service:
    • user data, like a username or id
    • guild information, like a name or id
    • data inputted from using application commands or invoking interactions

storage of data

  • the applications require the following information as a minimum to provide you with the service:
    • user data, ids only
    • guild information, ids only
  • the applications also store the following information, however these are optional and are not required to provide you with the service:
    • data inputted from using application commands or invoking interactions

data that isn't processed nor stored

  • the applications will not process the following sensitive information, as per discord's privileged intents:
    • user data, when profiles and statuses are updated
    • guild information, when a member is added/removed from a guild
    • guild information, contents of messages that do not @mention the application
  • the applications will not store the following sensitive information:
    • user information, content that is not related to the applications
    • guild information, content that is not related to the applications

usage of data

  • the data that is processed and stored is used for various purposes, such as:
    • to provide you with the service
    • to notify you about changes to the service
    • to allow you to participate with the interactive features with the service
    • to provide support with the service
    • to gather analytical data in order to improve the service
    • to monitor usage with the service
    • to prevent malicious use with the service

retention of data

  • your data will be retained in the service's databases for the duration of your discord account's lifespan in order to provide you with the service
    • this means that if your discord account were to be deleted, either through manual deletion or via discord's own inactive account deletion policy, your data associated with this app will be deleted too
    • data scheduled for deletion may take up to 7 days to be fully deleted
  • this data may further be stored for an extended period of time if required in order to comply with legal obligations
  • usage data will be anonymised and retained for internal analysis purposes; it may be kept for longer to improve the service over time

disclosure of data

  • all data related to the service are not disclosed to any third parties: they are only available within the scope of the service
  • separate applications may access data from another application in order to provide you with its services
  • the developers may disclose your personal data to discord officers if they believe it is necessary to, examples including:
    • violation of discord's terms of service
    • violation of discord's community guidelines
  • the developers may also disclose your personal data to other sources in order to:
    • comply with a legal obligation
    • protect against legal liability
    • protect the safety of users of the service
    • defend the rights of the developers

security of data

  • the security of your data are of the developers' upmost interests
  • however, it should be noted that no transmission over the internet is 100% secure and they do not guarantee security
  • the developers will strive to secure your data to a commercially acceptable level

your rights to your data

  • every user using the service has the following rights to their data:
    • the right to rectify and correct mistakes of your data
    • the right to request a copy of all your data the service has stored
    • the right to delete your data
  • these rights are available through usage of the services
  • alternatively, you can join the developers' discord support guild and ask for your rights to be upheld there

your agreement to this policy

  • when you use the service in any way, such as installing an application, you agree to all the terms listed in this policy
  • furthermore, if an application is authorised in a guild, the guild owner accepts these terms on behalf of all server members
  • if you do not agree with this policy:
    • as a guild member, you can leave the guilds which contain the applications
    • as a guild owner, you can revoke the service's application's integrations from the guild

changes to this policy

  • this policy is updated as necessary in order to make sure everything is up to date
  • it's also updated to stay compliant with the relevant laws related to discord and other privacy changes
  • every update to this policy is in effect as soon as it's accessible
  • you will be notified of changes to the privacy policy through usage of the service
  • you should also periodically review this privacy policy for any changes
